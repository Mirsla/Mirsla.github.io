SpringBoot关于thymeleaf引用问题

发表于 2017-10-31 | 分类于 Spring Boot | 阅读次数

在刚开始学习使用Spring Boot 做demo的时候遇到了一个关于thymeleaf引用静态资源的问题。这里记录下，可能会对以后能有点帮助，由于过去太久才写的这篇博客，可能会有些不完善的地方。

当时使用的环境是：Spring boot 1.5.6的版本，thymeleaf这些都是整套配置，编辑器是IDEA。

错误重现可能需要等一段时间了，最近太忙，没发。现在说说关于thymeleaf中的配置问题吧。

pom.xml基本引用

application.properties配置

其实使用默认配置也是可以的。这样的话就可以直接在项目中返回你需要到的html页面的路径和文件名就行了，一个String的字符串。刚开始使用的时候你会发现如果就是这样使用会非常的蛋疼，还有就是你明明关闭了缓存，为什么每次修改页面之后刷新会没有作用？

thymeleaf稍微高级点的东西

thymeleaf需要的其他jar包

  • thymeleaf缓存

现在先说第一个问题吧，为什么我明明关闭了thymeleaf的缓存，但是没有启任何作用呢？其实解决方式很简单，直接Ctrl+Shift+F9之后在刷型页面就起作用了，毕竟我也是一个彩笔，如果有更好的方式我希望你能告诉我。

  • thymeleaf闭合标签问题

thymeleaf需要闭合路径，也就是说我们正常的html页面有很多标签都需要在去手动修改的。必须要求闭合标签，如果写好静态页面之后，页面又比较大的话一个一个的去修改也是一件非常蛋疼的事情。

thymeleaf在2.0的时候这个好像是没有办法避免的事情，只有换一种方式，将thymeleaf的jar包进行升级，在pom文件中对thymeleaf的版本进行修改
